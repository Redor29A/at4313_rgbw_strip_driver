# AVR RGBW Controller

RGBW контроллер на `ATtiny4313` с энкодером, кнопкой, статусным светодиодом и дисплеем `TM1637`.

## Возможности

- Управление RGBW светодиодной лентой через PWM
- Два ручных пресета цвета
- Эффекты `RAIN`, `PULS`, `tEnP`
- UART-управление цветом с ПК
- Tkinter GUI для управления цветом и эффектами с компьютера
- Автоматическая запись настроек в EEPROM при изменении
- CRC-8 проверка EEPROM при старте
- Вывод `Err` на дисплей при поврежденной EEPROM
- Сборка отдельных `firmware.hex` и `firmware.eep`

## Железо

- MCU: `ATtiny4313`
- Частота: `20 MHz`
- Дисплей: `TM1637`
- Управление: энкодер с кнопкой
- Программатор в Makefile: `usbasp-clone`

| PB4 | PWM output |

## Управление

- Вращение энкодера без редактирования переключает параметр внутри текущего режима.
- Нажатие кнопки включает или выключает редактирование.
- В режиме редактирования значение выбранного параметра сразу записывается в EEPROM.
- Если выбран заголовок режима, вращение в режиме редактирования переключает режим.
- Status LED горит, когда включено редактирование.

## Режимы

| Экран | Режим | Параметры |
|---|---|---|
| `USb` | UART/off | Служебный режим, выходы выключены |
| `MAn1` | Manual 1 | `BRG`, `R`, `G`, `B`, `W` |
| `MAn2` | Manual 2 | `BRG`, `R`, `G`, `B`, `W` |
| `rAIn` | Rainbow | `SPD`, `DIR`, `BRG`, `SAT` |
| `PULS` | Pulse | `SPD`, `BRG`, `R`, `G`, `B`, `W` |
| `tEnP` | White temperature | `BRG`, `TMP` |

### Параметры

- `BRG` - яркость, обычно `0..100`
- `SPD` - скорость эффекта
- `DIR` - направление rainbow
- `SAT` - насыщенность rainbow
- `TMP` - температура белого от теплого к холодному

## UART

UART работает на `14400 baud`, формат `8N1`. ПК отправляет короткие бинарные пакеты:

```text
A5 cmd a b c d e xor
```

`xor` - XOR первых 7 байт.

Команды:

| cmd | Назначение | Payload |
|---|---|---|
| `0x01` | Direct RGBW control | `R G B W BRG` |
| `0x02` | Release UART control | не используется |

Команда `0x01` переводит прошивку в UART-control: на дисплее остается `USb`, а выходы управляются байтами из ПК без записи в EEPROM. Команда `0x02` отпускает UART-control, после чего снова работают локальные режимы меню.

## PC GUI

GUI находится в [tools/rgbw_control.py](tools/rgbw_control.py).

Зависимости для ПК:

- Python 3
- `tkinter`
- `pyserial`

Установка `pyserial`:

```bash
python3 -m pip install pyserial
```

Запуск:

```bash
python3 tools/rgbw_control.py
```

При первом запуске рядом со скриптом создается `tools/rgbw_config.json`. В нем сохраняются порт, baud rate, последний цвет, яркость и выбранный эффект.

GUI умеет:

- отправлять статический RGBW-цвет
- крутить `Rainbow` на ПК
- крутить `Pulse` на ПК
- управлять `White Temp` с ПК
- отпускать UART-control кнопкой `Release`

## EEPROM

Все настройки меню, лимиты, названия режимов и текущая позиция меню лежат в EEPROM. В RAM остаются только рабочие переменные эффектов и текущие индексы.

Последний байт EEPROM-структуры хранит CRC-8. После каждой записи настройки CRC пересчитывается. При старте прошивка проверяет CRC; если проверка не прошла, на дисплей выводится `Err`, а выполнение останавливается.

После изменения структуры меню нужно прошивать не только flash, но и EEPROM.

## Сборка

Зависимости:

- `avr-gcc`
- `avr-libc`
- `make`
- `avrdude`

Сборка:

```bash
make
```

Результаты:

- `build/firmware.hex` - программа для flash
- `build/firmware.eep` - начальные данные EEPROM

Прошивка:

```bash
make flash
```

Команда `make flash` прошивает и `firmware.hex`, и `firmware.eep`.

## Fuse Bits

Пример настройки fuse bits для внешнего генератора `20 MHz`:

```bash
avrdude -p t4313 -c usbasp-clone \
  -U lfuse:w:0xFF:m \
  -U hfuse:w:0xDF:m \
  -U efuse:w:0xFF:m
```
